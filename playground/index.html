1

Browser Contexts — Isolated Parallel Sessions

Create multiple isolated browser contexts in a single test. Each has its own cookies, localStorage, and session state — perfect for testing multi-user scenarios simultaneously.

Interactive Demo
Playwright Test
// Two users, same test, fully isolated
const alice = await browser.newContext();
const bob = await browser.newContext();

const pageA = await alice.newPage();
const pageB = await bob.newPage();

// Login as Alice
await pageA.fill('#ctx-username', 'alice');
await pageA.fill('#ctx-password', 'pass123');
await pageA.click('#ctx-login');

// Login as Bob (different context!)
await pageB.fill('#ctx-username', 'bob');
await pageB.fill('#ctx-password', 'pass456');
await pageB.click('#ctx-login');

// Verify isolation — each sees their own user
await expect(pageA.locator('#ctx-status'))
  .toContainText('alice');
await expect(pageB.locator('#ctx-status'))
  .toContainText('bob');
Why not Cypress?

Cypress runs inside a single browser — there's no concept of browser contexts. You cannot test two users simultaneously or verify session isolation in a single test. You'd need separate test files and can't compare state side by side.

Multi-Page — Tabs & Popups

Playwright can handle popups, new tabs, and cross-page communication. Click the link below — it opens a real popup that sends a message back.

Interactive Demo

Click to open a popup window:

Open Popup Window
Playwright Test
// Capture the popup when it opens
const [popup] = await Promise.all([
  page.waitForEvent('popup'),
  page.click('#popup-link'),
]);

// Interact with the popup
await expect(popup)
  .toHaveTitle('Popup Window');

// Click button in popup
await popup.click('#sendMessage');

// Verify parent received the message
await expect(page.locator('#popup-status'))
  .toContainText('Hello from popup!');
Why not Cypress?

Cypress automatically strips target="_blank" and cannot handle popups or new tabs at all. It's architecturally limited to a single page — there's no API to capture or interact with a second tab.

3

Network Routing — Intercept, Mock, Modify

Playwright's page.route() gives you full control over network requests: mock responses, modify request headers, and even abort requests entirely.

Interactive Demo
    Playwright Test
    // Mock the API response
await page.route('**/api/users', async route => {
  await route.fulfill({
    contentType: 'application/json',
    body: JSON.stringify([
      { name: 'Ada Lovelace', role: 'Engineer' },
      { name: 'Grace Hopper', role: 'Admiral' },
    ]),
  });
});

await page.click('#load-users');

// Verify mocked data rendered
await expect(page.locator('#users-list li'))
  .toHaveCount(2);

// Modify request headers
await page.route('**/*', async route => {
  await route.continue({
    headers: { ...route.request().headers(),
      'X-Custom': 'injected' }
  });
});

// Abort image requests
await page.route('**/*.png', route =>
  route.abort()
);
    Why not Cypress?

    Cypress's cy.intercept cannot modify outgoing request headers or request bodies before they're sent. It also cannot abort requests or do HAR-based replay. Playwright gives you full bidirectional control.

    Auto-Waiting — Actionability Checks

    Playwright automatically waits for elements to be visible, enabled, and stable before acting on them. No cy.wait(), no sleep(), no flaky retry wrappers.

    Interactive Demo
    I appeared!
    Waiting...
    Playwright Test
    // Element appears after 2s — no waitFor needed
await page.click('#show-delayed');
await expect(page.locator('#delayed-element'))
  .toBeVisible();

// Input enables after 1.5s — auto-waits
await page.click('#enable-input-btn');
await page.fill('#delayed-input', 'typed!');

// Text updates after 1s — assertion auto-retries
await page.click('#update-text-btn');
await expect(page.locator('#delayed-text'))
  .toHaveText('Updated!');

// Zero sleep calls. Zero flake.
    Why not Cypress?

    Cypress retries assertions but doesn't auto-wait for actionability on .type() or .click(). If an element is disabled or not yet visible, you often need explicit cy.wait() or custom retry logic. Playwright's actionability checks are built into every action.

    Device Emulation — Mobile, Geo, Permissions

    Playwright provides built-in device presets, geolocation overrides, permission grants, and full user agent spoofing — all configured per-context.

    Interactive Demo
    Viewport: — User Agent: — Color Scheme: — Geolocation: —
    Playwright Test
    import { devices } from '@playwright/test';

const iPhone = devices['iPhone 14 Pro'];

const context = await browser.newContext({
  ...iPhone,
  locale: 'en-US',
  geolocation: {
    latitude: 37.7749,
    longitude: -122.4194,
  },
  permissions: ['geolocation'],
  colorScheme: 'dark',
});

const page = await context.newPage();
await page.goto('/');

// Viewport matches iPhone 14 Pro
const vp = page.viewportSize();
expect(vp.width).toBe(393);

// Geolocation was granted
await page.click('#emu-geo-btn');
await expect(page.locator('#emu-geo'))
  .toContainText('37.77');
    Why not Cypress?

    Cypress offers basic cy.viewport() but has no geolocation override, no permission grants, no user agent spoofing, and no device presets. Testing mobile experiences or location-based features requires workarounds or plugins.

    Tracing — Time-Travel Debugging

    Playwright's trace viewer captures DOM snapshots, screenshots, network activity, and console logs at every step — giving you time-travel debugging for any test failure.

    What a Trace Captures
    📸 Screenshots
    🖼 DOM Snapshots
    🌐 Network
    💻 Console
    📄 Source

    View with: npx playwright show-trace trace.zip

    Playwright Test
    // Start tracing before actions
await context.tracing.start({
  screenshots: true,
  snapshots: true,
  sources: true,
});

// ... perform test actions ...
await page.goto('/');
await page.click('#ctx-login');
await page.fill('#ctx-username', 'trace-user');

// Stop and save the trace
await context.tracing.stop({
  path: 'test-results/trace.zip',
});

// Open: npx playwright show-trace trace.zip
// See DOM, network, console at every step
    Why not Cypress?

    Cypress captures screenshots and video, but has no interactive trace viewer. You can't inspect DOM state, network requests, or console logs at each individual step. Playwright's trace viewer gives you a full time-travel debugging experience.
